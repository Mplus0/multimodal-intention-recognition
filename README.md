# Multimodal User Intention Recognition

Machine Learning course project, 2025-2026 Spring semester.

This repository implements an end-to-end multimodal user intention recognition pipeline for AR interaction scenarios. It refactors the teacher-provided feature-file workflow into a project-relative pipeline and adds experiments for clean data, modal noise, missing modalities, and an improved robustness-oriented fusion model.

Current repository status:

- The code pipeline and experiment entry points are complete and runnable on the course server.
- The repository does not contain the large raw dataset, local pretrained models, trained checkpoints, or official result files.
- Official training and evaluation have not been completed in this repository snapshot.
- `results/`, `figures/`, and `report/screenshots/` are output locations prepared for later server runs.

---

## 1. Project Requirements

The official task requires:

1. Train on user A and user B.
2. Test on user C.
3. Refactor the original workflow into:

```text
raw data -> preprocessing / feature extraction -> multimodal fusion model -> intention label
```

4. Build modal noise baselines by adding 20%, 40%, and 60% noise to each single modality.
5. Build missing-modality baselines by dropping any one modality and any two modalities.
6. Improve the model with a new module or loss term for robustness under noise and missing modalities.
7. Save logs, metrics, predictions, figures, screenshots, and references for the report and defense.

## 2. Formal Modalities and Dataset Split

Formal modality keys:

```text
imu
gesture
audio
text
scene
```

Feature shape contract:

| Modality | Shape | Meaning |
|---|---|---|
| `imu` | `(N, 10, 12)` | IMU sequence features |
| `gesture` | `(N, 10, 768)` | gesture visual sequence features |
| `audio` | `(N, 10, 39)` | MFCC audio sequence features |
| `text` | `(N, 10, 384)` | ASR / sentence embedding sequence features |
| `scene` | `(N, 768)` | scene visual feature |

Dataset split:

| Split | Users |
|---|---|
| Train | `user_A`, `user_B` |
| Test | `user_C` |

The sample index is generated by `src/data/build_samples.py`. For the full dataset, the expected video-level summary is:

```text
sample_count: 39
train: 26
test: 13
user_A: 13
user_B: 13
user_C: 13
```

## 7. Improved Model Summary

The improved model is:

```text
Reliability-Gated Multimodal Fusion with Modality Dropout
```

Main ideas:

1. Each modality has a reliability gate.
2. `modality_mask` is explicitly used during fusion.
3. Missing modalities are assigned zero contribution.
4. Training uses random modality dropout to improve robustness under missing modalities.

Key files:

```text

src/data/improved_transforms.py
src/training/improved_experiment_runner.py
```

## 8. Quick Start on the Course Server

Enter the project:

```bash
cd /share/home/tm1078571822880000/a1086482920/Mplus0/multimodal-intention-recognition
conda activate machine_learning
export PY=/share/home/tm1078571822880000/a1086482920/Mplus0/software/miniconda3/envs/machine_learning/bin/python
mkdir -p report/screenshots report/terminal_logs
```

Check Python / CUDA:

```bash
$PY -V
$PY -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.version.cuda); print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'CPU')"
```

Build sample index:

```bash
$PY src/data/build_samples.py --config configs/default.yaml --output data/processed/sample_index.json
```

Build or inspect features:

```bash
$PY src/data/build_features.py --config configs/default.yaml --metadata-json data/processed/sample_index.json --build-missing --limit 1
$PY src/data/build_features.py --config configs/default.yaml --metadata-json data/processed/sample_index.json --build-missing --limit 3
$PY src/data/build_features.py --config configs/default.yaml --metadata-json data/processed/sample_index.json --build-missing --limit 100000
```

Run clean baseline:

```bash
$PY experiments/train.py --config configs/default.yaml
$PY experiments/test.py --config configs/default.yaml --checkpoint results/checkpoints/best.pt
```

Run baseline experiment groups:

```bash
$PY experiments/run_clean_baseline.py --config configs/default.yaml
$PY experiments/run_noise_baseline.py --config configs/noise.yaml --base-config configs/default.yaml
$PY experiments/run_missing_baseline.py --config configs/missing_modality.yaml --base-config configs/default.yaml
```

Run improved-model checks and experiments:

```bash
$PY src/models/improved_model.py --config configs/default.yaml --smoke-test
$PY src/data/improved_transforms.py
$PY experiments/run_improved_model.py --config configs/improved_model.yaml --base-config configs/default.yaml --mode clean --smoke-test --epochs 1
$PY experiments/run_improved_model.py --config configs/improved_model.yaml --base-config configs/default.yaml --mode clean --max-samples 30 --epochs 1
$PY experiments/run_improved_model.py --config configs/improved_model.yaml --base-config configs/default.yaml --mode clean
$PY experiments/run_improved_model.py --config configs/improved_model.yaml --base-config configs/default.yaml --mode ablation
$PY experiments/run_improved_model.py --config configs/improved_model.yaml --base-config configs/default.yaml --mode noise
$PY experiments/run_improved_model.py --config configs/improved_model.yaml --base-config configs/default.yaml --mode missing
```
